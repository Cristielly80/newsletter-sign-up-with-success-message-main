# Frontend Mentor - Formulário de inscrição na newsletter com mensagem de sucesso

![Prévia do design do desafio](./design/desktop-design.jpg)

## Bem-vindo! 👋

Obrigado por conferir este desafio de front-end.

Os desafios do Frontend Mentor ajudam você a melhorar suas habilidades de programação construindo projetos realistas.

**Para fazer este desafio, você precisa ter um conhecimento básico de HTML, CSS e JavaScript.**

## O desafio

Seu desafio é construir este formulário de newsletter e deixá-lo o mais próximo possível do design.

Você pode usar quaisquer ferramentas que quiser para completar o desafio. Se quiser praticar algo específico, fique à vontade!

Seus usuários devem ser capazes de:

- Adicionar o e-mail e enviar o formulário  
- Ver uma mensagem de sucesso com o e-mail após o envio bem-sucedido  
- Ver mensagens de validação do formulário se:
  - O campo estiver vazio  
  - O e-mail não estiver em um formato válido  
- Visualizar o layout ideal dependendo do tamanho da tela do dispositivo  
- Ver estados de hover e foco em todos os elementos interativos  

Quer ajuda com o desafio? [Entre na comunidade](https://www.frontendmentor.io/community) e faça perguntas no canal **#help**.

## Onde encontrar tudo

Sua tarefa é construir o projeto com base nos designs dentro da pasta `/design`.  

Você encontrará versões para mobile e desktop.

Os designs estão em formato JPG estático. Isso significa que você precisará usar seu próprio julgamento para definir estilos como `font-size`, `padding` e `margin`.

Se quiser acessar os arquivos de design (Sketch ou Figma), você pode [se tornar um membro PRO](https://www.frontendmentor.io/pro).

Todos os arquivos necessários para este projeto estão na pasta `/assets`. As imagens já estão exportadas no tamanho correto e otimizadas.

Também incluímos arquivos de fontes variáveis e estáticas. Você pode usar Google Fonts ou hospedar localmente.

Existe também um arquivo `style-guide.md` com informações como paleta de cores e fontes.

## Uso de assistentes de IA

## Construindo seu projeto

Sinta-se à vontade para usar o fluxo de trabalho que preferir. Abaixo está um processo sugerido:

1. Inicialize seu projeto como um repositório público no [GitHub](https://github.com/). Isso facilita compartilhar seu código.
2. Configure o repositório para publicar seu projeto online.
3. Analise os designs antes de começar para planejar suas classes CSS.
4. Estruture o conteúdo com HTML antes de estilizar.
5. Defina estilos base como `font-family` e `font-size`.
6. Estilize começando do topo da página para baixo.
